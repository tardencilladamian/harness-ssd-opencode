# UI System

## Purpose

Define visual, UX, and accessibility rules.

## Design Principles

- Build the actual usable workflow first.
- Keep layouts responsive.
- Make common actions obvious.
- Prefer accessible components.
- Avoid decorative UI that hides the task.

## Accessibility

- Keyboard navigation must work.
- Forms require labels and validation messages.
- Color contrast must be acceptable.
- Error states must be announced clearly.

## Browser Verification

For frontend work, define whether Playwright or another browser test tool is required.
