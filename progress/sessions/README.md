# Session Logs

Use this folder for session-level summaries.

Recommended filename:

```text
<DATE>-<FEATURE_ID>.md
```

Each session log should include:

- Date.
- Feature worked.
- Model or profile used when known.
- Commands executed.
- Files changed.
- Tests or verification run.
- Evidence created.
- Blockers.
- Next action.

Session logs do not replace:

- `progress/current.md`
- `progress/features/<FEATURE_ID_SLUG>.md`
- `progress/history.md`

They provide more granular continuity when a project has many AI-assisted sessions.
